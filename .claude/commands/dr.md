# /draft — 블로그 포스트 초안 생성

노션 포스트 DB에 새 초안 페이지를 만듭니다.
사용자에게 필요한 정보를 순서대로 묻고, 입력이 완료되면 노션에 페이지를 생성합니다.

## 실행 절차

### 1단계 — 정보 수집

AskUserQuestion 도구로 아래 항목을 **하나씩 차례로** 질문합니다.
각 질문은 이전 답변을 받은 뒤에 다음 질문을 합니다.

**질문 순서:**

1. **제목** — "포스트 제목을 입력해 주세요."
2. **유형** — "글 유형을 선택해 주세요. (기술 / 회고 / 사담 / 작업)"
3. **태그** — "태그를 쉼표로 구분해서 입력해 주세요. 없으면 엔터."
4. **설명** — "포스트 한 줄 설명(미리보기 문구)을 입력해 주세요. 없으면 엔터."

유형 입력값을 카테고리에 매핑합니다:
- 기술 / 회고 / 사담 → 카테고리: `기록`
- 작업 → 카테고리: `작업`

### 2단계 — 노션 DB ID 확인

`.env.local` 파일에서 `NOTION_DATABASE_POST_LINK` 값을 읽어 DB ID로 사용합니다.

### 3단계 — 유형별 본문 템플릿 선택

유형에 따라 아래 children 구조를 사용합니다.

---

**기술** (서비스 기획자/PM 관점의 개념 + 경험 아티클):
```
## 서론
이 개념을 접하게 된 계기, 또는 실무에서 마주친 문제 상황
→ 독자가 "나도 이런 상황 있었는데" 하고 공감할 수 있도록

## 본론

### 개념 정리
핵심 개념을 PM 언어로 풀어서 설명
(기술 구현보다 '왜 필요한가', '어떻게 쓰이는가' 중심으로)

### 실무에서 이렇게 썼다
직접 기획하거나 의사결정에 활용한 사례
(어떤 상황이었고, 어떤 판단을 했고, 결과는 어땠는지)

## 결론
이 개념에서 건진 인사이트 또는 앞으로 어떻게 적용할 건지
```

---

**회고**:
```
## 서론
이 회고를 쓰는 이유, 돌아보는 기간이나 주제

## 본론

### 한 것들
이 기간에 실제로 진행한 일, 의사결정, 협업

### 잘 됐던 것
의도한 대로 흘러간 것, 좋은 반응을 얻은 것

### 아쉬운 것
놓쳤거나 더 잘할 수 있었던 것, 반복되는 패턴이 있다면

## 결론
다음에 달라질 것, 또는 이 회고를 통해 정리된 생각
```

---

**사담**:
```
## 서론
이 글을 쓰게 된 계기, 요즘 어떤 생각을 하고 있었는지

## 본론
하고 싶은 말

## 결론
마무리 한마디, 또는 독자에게 던지고 싶은 질문
```

---

**작업** (프로젝트 기록, PM 관점):
```
## 서론
이 프로젝트를 시작하게 된 배경
어떤 문제를 해결하려 했는지

## 본론

### 내가 맡은 것
기획, 설계, 협업 등 실제로 담당한 영역

### 주요 의사결정
왜 이렇게 설계했는지, 어떤 트레이드오프가 있었는지

### 어려웠던 점과 해결
예상 못 한 문제와 그걸 어떻게 풀었는지

## 결론
결과 요약, 이 프로젝트에서 얻은 것
```

---

### 4단계 — 노션 페이지 생성

`mcp__notionApi__API-post-page` 도구로 페이지를 생성합니다.

- **parent**: `{ "database_id": "<NOTION_DATABASE_POST_LINK 값>" }`
- **properties**:
  - `제목`: title 타입 → 입력받은 제목
  - `카테고리`: select 타입 → 매핑된 카테고리 (`기록` 또는 `작업`)
  - `날짜`: date 타입 → 오늘 날짜 (ISO 8601)
  - `태그`: multi_select 타입 → 입력받은 태그 (없으면 빈 배열)
  - `설명`: rich_text 타입 → 입력받은 설명 (없으면 빈 문자열)
  - `상태`: status 타입 → `"백로그"` (고정)
- **children**: 3단계에서 선택한 유형별 템플릿

### 5단계 — 결과 안내

생성 성공 시:
- 생성된 페이지 제목과 노션 URL을 출력합니다.
- 상태가 "백로그"로 설정되어 있음을 알립니다.

생성 실패 시:
- 오류 메시지를 보여주고, DB ID나 권한을 확인하도록 안내합니다.

## 주의사항

- 태그나 설명을 입력하지 않으면 빈값으로 처리합니다 (에러 아님).
- 날짜는 오늘 날짜를 자동으로 설정합니다. 사용자에게 묻지 않습니다.
- 본문에 `—` (em dash)를 절대 사용하지 않습니다. 대신 쉼표, 마침표, 줄바꿈 등으로 대체합니다.
