# /ed — 블로그 포스트 본문 수정

노션 포스트의 본문 내용을 수정합니다.
제목으로 글을 검색한 뒤, 원하는 섹션의 내용을 교체합니다.

## 실행 절차

### 1단계 — 수정할 글 찾기

AskUserQuestion 도구로 아래를 순서대로 질문합니다.

1. **제목** — "수정할 포스트 제목을 입력해 주세요. (일부만 입력해도 됩니다)"

`mcp__notionApi__API-post-search` 도구로 검색합니다.
- 검색 결과가 여러 개면 목록을 보여주고 어떤 글인지 다시 확인합니다.
- 결과가 없으면 "찾지 못했어요. 제목을 다시 확인해 주세요."라고 안내합니다.

### 2단계 — 현재 본문 확인

`mcp__notionApi__API-get-block-children` 도구로 해당 페이지의 블록을 가져옵니다.
현재 본문 구조(헤딩 목록)를 사용자에게 보여줍니다.

예시 출력:
```
현재 섹션 구조:
1. 서론
2. 본론 > 개념 정리
3. 본론 > 실무에서 이렇게 썼다
4. 결론
```

### 3단계 — 수정 내용 확인

2. **수정할 섹션** — "어떤 섹션을 수정하시겠어요? (번호 또는 섹션 이름)"
3. **새 내용** — "새로 들어갈 내용을 입력해 주세요."

### 4단계 — 본문 수정

기존 내용은 삭제하지 않고, 아래 순서로 블록을 추가합니다.

`mcp__notionApi__API-patch-block-children`으로 해당 섹션 헤딩 바로 아래에 다음 순서로 삽입합니다:

1. 새로 작성한 내용 블록
2. `divider` 블록 (구분선)
3. 기존 내용 블록은 그대로 구분선 아래에 남아있음 (건드리지 않음)

결과 구조:
```
## 섹션명
[새로 작성한 내용]
---
[기존 내용 — 보존]
```

헤딩 블록 자체는 건드리지 않습니다.

### 5단계 — 결과 안내

수정 성공 시:
- "'{섹션명}' 섹션이 수정됐어요. 기존 내용은 구분선 아래에 보존돼 있어요."라고 안내합니다.
- 노션 페이지 URL을 함께 출력합니다.

수정 실패 시:
- 오류 내용을 보여주고 다시 시도하도록 안내합니다.

## 주의사항

- 기존 내용은 절대 삭제하지 않습니다. 구분선 아래에 항상 보존합니다.
- 헤딩 블록은 건드리지 않습니다.
- 속성(제목, 태그, 카테고리 등) 수정은 이 커맨드 범위 밖입니다.
- 새로 작성하는 내용에 `—` (em dash)를 절대 사용하지 않습니다. 대신 쉼표, 마침표, 줄바꿈 등으로 대체합니다.
